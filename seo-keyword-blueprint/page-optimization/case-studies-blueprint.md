# Case Studies Blueprint

## Page Goal

Turn proof into a ranking and conversion asset. Case studies should help the site rank for proof-intent terms and help warm buyers commit.

## Case Study Hub

- `URL`: `/work`
- `Primary keyword`: `digital marketing case studies`
- `Secondary`: `seo case study`, `google ads case study`, `marketing automation case study`
- `H1`: `Growth Work and Measurable Results`

## Priority Case Study Pages

| URL | Primary keyword | Core proof angle | Main CTA |
|---|---|---|---|
| `/work/seo-growth` | `seo case study` | rankings, sessions, authority growth | Book an SEO strategy call |
| `/work/paid-media` | `google ads case study` | ROAS improvement, tracking cleanup, CAC reduction | Request an ad account review |
| `/work/automation` | `marketing automation case study` | hours saved, response speed, process reliability | Start an automation scoping call |

## Shared Content Outline

1. Client context and starting state
2. The bottleneck or failure pattern
3. Strategy and why it was chosen
4. Execution details
5. Results with numbers
6. What changed operationally
7. Key takeaways
8. CTA tied to the same service

## Heading Structure

- `H2`: The problem
- `H2`: The strategy
- `H2`: What was implemented
- `H2`: Results
- `H2`: What this means for similar businesses

## Internal Links To Add

- Matching service page
- One or two relevant blogs
- Hub page `/work`
- Book or contact page

## Trust Elements To Include

- Timelines
- Before-and-after metrics
- Tool/process detail
- Client role or testimonial
- Screenshots where allowed

## Conversion Elements To Include

- CTA after the results block
- CTA after takeaways
- Service-specific offer, not a generic contact link only

## SEO Notes

- Keep the keyword in the title, intro, and one subheading
- Use descriptive slugs and explicit result language in titles
- Add `Article` or `CreativeWork` schema only if it reflects the final implementation better than a generic page

## Common Mistakes To Avoid

- Writing vague portfolio summaries instead of real narratives
- Hiding the numbers
- Linking only back to the work hub instead of the matching service
