# Internal Linking Plan

## Main Principle

Every new page should both receive links from an existing hub and send visitors toward a conversion path. No page should sit alone.

## Link Quotas By Page Type

| Page type | Links out | Links in target |
|---|---:|---|
| Homepage | 8-12 | Receives nav, footer, branded links |
| Service page | 5-8 | Homepage, services hub, relevant blogs, case studies |
| Sub-service page | 4-6 | Parent service, blogs, tool, case studies |
| Blog post | 3-5 | Writing hub, service page, adjacent blog |
| Case study | 3-5 | Work hub, service page, related blogs |
| Tool page | 4-6 | Homepage, SEO page, blogs, footer |
| About / contact / book | 3-5 | Nav, footer, homepage, service pages |

## Required Link Paths

- Homepage -> all four services
- Homepage -> `/seo-tools`
- Homepage -> top three case studies
- Services hub -> each service page
- Each service page -> one matching case study
- Each service page -> two matching blogs
- Each case study -> matching service page
- Each blog -> one service page and one case study
- Tool page -> SEO service page and contact / booking page

## Recommended Anchor Style

- Use descriptive anchors such as `SEO consultant`, `Google Ads audit`, `n8n automation`, `see the SEO case study`
- Avoid repeated exact-match anchors in the same page footer or CTA zone
- Vary anchors naturally with service or outcome modifiers

## Navigation Recommendations

- Keep the top nav simple
- Add footer links to the main services, tool page, work hub, and writing hub
- Add contextual links inside service intros and case-study summaries, not just buttons

## Priority Internal Linking Rollout

1. Homepage -> services / tool / work
2. Services -> matching case studies
3. Blogs -> services + proof pages
4. Tool -> SEO audit service + booking
5. Case studies -> service pages + contact

## Internal Linking Mistakes To Avoid

- Linking every blog to every service
- Using button-only internal links without contextual links in body copy
- Leaving the tool page disconnected from paid services
- Sending proof pages only to the homepage
