# Metadata Plan

## Rules

- Keep titles under roughly 60 characters when possible
- Keep meta descriptions around 140-160 characters
- Lead with the keyword, then the payoff
- Use one clear CTA or value proposition in the description

## Title Patterns

| Page type | Title pattern |
|---|---|
| Homepage | `Digital Marketing Consultant in Dallas | SEO, Ads, Automation` |
| Services hub | `Digital Marketing Services | SEO, Google Ads, Automation` |
| Service page | `{Primary keyword} | {Main benefit} | Shree Krishna Gauli` |
| Sub-service page | `{Primary keyword} | {Specific deliverable}` |
| Blog post | `{Primary keyword}: {Clear payoff}` |
| Case study | `{Main result} | {Service} Case Study` |
| Tool page | `Free SEO Audit Tool | Check Titles, H1s, Speed, Schema` |

## Meta Description Patterns

| Page type | Meta pattern |
|---|---|
| Homepage | `Work directly with Shree Krishna Gauli on SEO, Google Ads, and marketing automation systems built for measurable growth.` |
| Service page | `Get {service outcome} through {process or deliverables}. Direct consultant-led support, clear reporting, and measurable next steps.` |
| Blog post | `Learn how to {solve the problem} with a practical framework, examples, and when to get outside help.` |
| Case study | `See how {problem} was fixed through {service} and what results changed.` |
| Tool page | `Audit any page for titles, meta descriptions, H1s, schema, canonical tags, and SEO issues in minutes.` |

## Metadata Mistakes To Avoid

- Repeating the brand name before the keyword on every page
- Using the same description template across all service pages
- Stuffing city modifiers into titles that are not local-intent pages
- Writing titles that describe the page but do not promise a benefit
