# Topic Cluster Plan

## Why Clusters Matter

The idea behind topic clusters is simple, even if the execution takes discipline: one big pillar post targets a broad keyword, and 3–6 smaller supporting posts target more specific long-tail keywords. They all link to each other. Google sees the cluster as a signal that your site has comprehensive coverage of the topic, not just one random article.

Each cluster below ends at a service page — that's the conversion destination. The blog posts educate, the service page sells. Keep that flow in mind.

## Cross-Cluster Linking Rules

1. Each pillar post must link to at least 1 other pillar post
2. Supporting posts should link to their own pillar + 1 supporting post from another cluster
3. Every cluster must link to at least 1 case study
4. Every post links to at least 1 service page
5. Anchor text must be descriptive and varied (no exact-match keyword stuffing)

## Content Production Priority

| Cluster | Priority | Reason |
|---------|----------|--------|
| Cluster 1: SEO | Highest | Core service, most keywords, existing content base |
| Cluster 2: Paid Media | High | Direct revenue service, several existing posts |
| Cluster 3: Automation | High | Key differentiator, growing search interest |
| Cluster 4: Growth Strategy | Medium | Broad funnel, supports all services |
| Cluster 6: Lead Generation | Medium | Cross-service, high commercial intent |
| Cluster 5: Social Media | Medium | Service line support, lower priority |
| Cluster 7: Local SEO | Lower initially | Requires more authority, industry-specific |
