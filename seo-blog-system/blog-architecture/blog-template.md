# Blog Template

## The Standard Template

Every blog post — whether it's generated by AI or written by hand — should follow this general structure. Not rigidly. A checklist post can be shorter and punchier. A pillar post might need more sections. But this is the baseline, and deviating from it should be a deliberate choice, not an accident.

---

## Required Fields (Prisma Post Model)

```
title:          String (H1, max 70 chars preferred)
slug:           String (URL slug, lowercase-hyphenated, max 60 chars)
excerpt:        String (150–200 chars, used for cards and social sharing)
content:        String (full HTML/Markdown body)
seoTitle:       String (50–60 chars, for <title> tag)
metaDesc:       String (150–160 chars, for meta description)
canonical:      String (https://shreegauli.com/blogs/{slug})
coverImage:     String (URL to cover image, optional)
coverImageAlt:  String (alt text for cover image)
publishedAt:    DateTime (null for drafts, set on publish)
categories:     Category[] (at least 1)
tags:           Tag[] (3–5 recommended)
author:         Author (Shree Krishna Gauli)
```

## Content Structure Template

```markdown
# {H1: Post Title — Keyword Optimized}

{Introduction: 2–3 paragraphs}
- Hook the reader with a relatable problem or bold statement
- State what the post will cover
- Briefly explain why it matters to the target audience

## {H2: First Major Section}

{Content: 2–4 paragraphs with practical value}
- Include specific examples, data points, or steps
- Internal link opportunity: link to a relevant service page or blog post

### {H3: Sub-section if needed}

{Supporting detail, example, or breakdown}

## {H2: Second Major Section}

{Content: 2–4 paragraphs}
- Continue delivering value
- Include another internal link

## {H2: Third Major Section}

{Content: 2–4 paragraphs}
- Include a practical example or mini case study
- Link to a case study if relevant: /work/[relevant-slug]

## {H2: Fourth Major Section (optional)}

{Additional depth if the post warrants it}

---

**{Inline CTA}**

{One-liner connecting the post topic to a service}
[{CTA Button Text}]({CTA URL})

---

## {H2: Key Takeaways / Conclusion}

{2–3 paragraphs summarizing the post}
- Restate the main points
- Give a clear next step
- Close with confidence, not fluff

---

{End-of-Post CTA Block}

### Ready to {outcome relevant to post topic}?

{One sentence value proposition tied to the post's theme.}

[Book a Strategy Call →](/lp/book-a-call)  |  [Run a Free SEO Audit →](/seo-tools)
```

## SEO Title Patterns

These patterns aren't rules, they're starting points. The best title is the one that matches what the searcher expects to find while also making them want to click. Sometimes that means breaking the pattern.

| Post Type | Pattern | Example |
|-----------|---------|---------|
| How-to | How to {Action}: {Modifier} | How to Do a Google Ads Audit: A Step-by-Step Guide |
| Checklist | {Topic} Checklist for {Audience} | Technical SEO Audit Checklist for Service Businesses |
| Comparison | {A} vs {B}: {Question or Angle} | SEO vs Google Ads: What to Fix First |
| Listicle | {Number} {Items} for {Audience/Goal} | 15 Questions to Ask Before Hiring a Marketing Agency |
| Guide | {Topic} Guide for {Audience} ({Year}) | Marketing Automation Guide for Small Business (2026) |
| Problem-Solution | Why {Problem} and How to Fix It | Why Most n8n Workflows Break in Production |

## Meta Description Patterns

The meta description is your pitch in the search results. Google sometimes rewrites it, but a good one sticks more often than not.

| Post Type | Pattern |
|-----------|---------|
| How-to | Learn how to {action} with this step-by-step guide. Includes {specifics} for {audience}. |
| Checklist | Use this {topic} checklist to {outcome}. Covers {key areas} for {audience}. |
| Comparison | {A} or {B}? Compare {aspects} to decide the best {goal} strategy for your business. |
| Listicle | {Number} {items} every {audience} should know. From {topic A} to {topic B}. |
| Guide | Complete guide to {topic} for {audience}. Learn {benefit 1}, {benefit 2}, and {benefit 3}. |

## Image Requirements

Images aren't strictly necessary for every post, but a cover image helps with social sharing and makes the blog hub look better.

### Cover Image
- **Size:** 1200×630px (OG-compatible)
- **Format:** WebP preferred, JPG acceptable
- **Alt text:** Descriptive, include keyword if natural
- **Naming:** `{slug}-cover.webp`

### In-Content Images (if any)
- **Alt text:** Required, descriptive
- **Lazy loading:** Enabled by default in Next.js
- **Compression:** Optimize before upload

## Category Assignment

One category per post. Pick the one that best matches the searcher's intent, not the one that has the most keywords. Use tags for secondary topics.

| Post Topic | Category | Tag Examples |
|-----------|----------|-------------|
| SEO, keyword research, audits, rankings | SEO | technical-seo, keyword-research, small-business |
| Google Ads, Meta Ads, PPC, ROAS | Paid Media | google-ads, meta-ads, small-business |

## Quality Checklist

Run through this before hitting publish. It takes five minutes and catches the things that are easy to miss.

- [ ] Title is under 70 characters and includes primary keyword
- [ ] SEO title is 50–60 characters
- [ ] Meta description is 150–160 characters
- [ ] Slug is under 60 characters, lowercase, hyphenated
- [ ] Content is at least 1,200 words (2,000+ for pillar posts)
- [ ] At least 1 internal link to a service page
- [ ] At least 2 internal links to related blog posts
- [ ] At least 1 CTA (inline and/or end-of-post)
- [ ] Cover image has alt text
- [ ] At least 1 category assigned
- [ ] 3–5 tags assigned
- [ ] No orphaned headings (H3 without parent H2)
- [ ] No duplicate content from other posts on the site
- [ ] **Reads well aloud** — natural tone, not robotic. This is the one people skip, and it matters the most.
- [ ] Provides actionable value, not generic advice
